API Documentation

DCF Valuation API

Request a discounted cash flow valuation for any supported US stock with your own assumptions in a single GET request. The response includes every intermediate number — base financials, year-by-year projections, discount factors, terminal value — so you can audit the math down to the dollar.

GET /v1/valuations/{ticker}

You supply the assumptions; we supply the financials. Company fundamentals (revenue, EBIT, D&A, capex, working capital, debt, share count) are fetched from our data provider, normalized into a canonical schema, and combined with your inputs by a deterministic valuation engine. Identical requests return identical results for a given model_version and source period.

v1 scope: non-financial US large caps. Banks and insurers are rejected with 422 — a standard free-cash-flow DCF doesn't apply to financial balance sheets.

Quickstart

Value Apple with a 9% discount rate, 2.5% perpetual growth, 30% EBIT margin, and a five-year growth ramp from 8% down to 4%:

Request
curl "https://api.example.com/v1/valuations/AAPL?\
wacc=0.09&\
terminal_growth=0.025&\
ebit_margin=0.30&\
revenue_growth=0.08,0.07,0.06,0.05,0.04&\
projection_years=5"

All rates are decimals: 0.09 means 9%. revenue_growth takes a single value applied to every year, or one comma-separated value per projection year. Prefer not to hand-write URLs? Use the endpoint builder below.

Authentication

Requests are authenticated with an API key sent in the X-API-Key header:

curl -H "X-API-Key: YOUR_KEY" "https://api.example.com/v1/valuations/AAPL?..."
Preview: per-customer API keys and usage metering are not yet enforced in the current release. The header is accepted and will become mandatory before general availability — include it now so your integration doesn't break later.

Endpoint builder

Enter your assumptions and get a ready-to-use endpoint URL. Values you'd quote as percentages go in as percentages here — the builder converts them to the decimal form the API expects, validates them against the same rules the server enforces, and emits parameters in canonical order (canonically-ordered URLs are HTTP-cacheable and dedupe cleanly).

Get a valuation

GET /v1/valuations/{ticker}

Runs a DCF for one ticker using the supplied assumptions and the company's most recent annual financials. The ticker is case-insensitive.

Query parameters

ParameterTypeDescription
waccdecimalrequired Discount rate. 0.09 = 9%. Must be positive and greater than terminal_growth.
terminal_growthdecimalrequired Perpetual growth rate for the Gordon-growth terminal value.
ebit_margindecimalrequired Operating margin applied to every projected year. 0.30 = 30%.
revenue_growthdecimal | listrequired Single value applied to all years (0.05) or comma-separated per-year values (0.08,0.07,0.06,0.05,0.04). A list must have exactly projection_years entries. Each value within ±50%.
tax_ratedecimaldefault 0.21 Effective tax rate applied to EBIT. Must not be negative.
projection_yearsintegerdefault 5 Explicit forecast horizon. 3–15.
sensitivitybooleandefault true Include a 3×3 sensitivity grid around your assumptions (WACC ±1% × terminal growth ±0.5%). Pass false to omit it.

Response schema

Every number the engine used or produced is returned, so the valuation can be rebuilt in a spreadsheet line by line. Monetary values are raw US dollars.

FieldDescription
model_versionVersion of the valuation engine that produced this result. Pin this when comparing valuations over time.
base_financialsThe normalized inputs: revenue, EBIT, D&A, capex, ΔNWC, net debt, diluted shares, current price — with source_period identifying the fiscal period they came from.
assumptionsYour assumptions, fully resolved: a scalar revenue_growth is echoed back expanded to one value per year, and defaults you omitted are filled in. What you see is exactly what the engine used.
projections[]Per year: revenue, ebit, fcf (unlevered free cash flow), discount_factor, and pv_fcf.
terminal_value / pv_terminal_value Gordon-growth terminal value and its present value.
enterprise_valueSum of all pv_fcf plus pv_terminal_value.
equity_valueEnterprise value minus net debt.
intrinsic_value_per_shareEquity value divided by diluted shares.
current_price / upside_pctLatest market price and the implied difference between it and intrinsic value, in percent.
sensitivityPer-share value across a 3×3 grid: rows are wacc_values (your WACC ±1%), columns are terminal_growth_values (±0.5%). The center cell equals intrinsic_value_per_share. Cells where the math is undefined (growth ≥ WACC) are null. Omitted when sensitivity=false.

Validation & errors

Assumption errors return 422 with the offending field named, so they can be surfaced directly against the right input in your UI:

422 Unprocessable Entity
{
  "detail": [
    { "field": "terminal_growth", "message": "must be less than wacc (Gordon growth formula)" }
  ]
}
StatusWhen
200Valuation computed.
404No valuation available for this ticker — either the symbol doesn't exist, or it falls outside the supported universe (non-financial US large caps). Retrying won't change the result.
422Invalid assumptions (terminal_growth ≥ wacc, projection_years outside 3–15, growth beyond ±50%, negative tax rate, malformed or wrong-length revenue_growth) — or a financial-sector ticker, which this model doesn't support.
502Provider data for this ticker couldn't be normalized. Not retryable; contact support.
503Upstream data provider unavailable. Retry with backoff.

Health check

GET /health

Returns 200 with the service status and current model_version. Unauthenticated; suitable for load-balancer probes.

Methodology

The engine projects unlevered free cash flow over the forecast horizon:

FCF = EBIT × (1 − tax_rate) + D&A − capex − ΔNWC

Revenue grows by your per-year rates; EBIT applies your margin to each year's revenue; D&A, capex, and ΔNWC scale with revenue at the base year's observed ratios. Cash flows are discounted at your WACC with end-of-year timing. Terminal value uses Gordon growth off the final projected year:

TV = FCF_final × (1 + terminal_growth) / (wacc − terminal_growth)

Equity value is the present value of the projected cash flows plus the present value of terminal value, minus net debt; per-share value divides by diluted shares outstanding.

DCF outputs are highly sensitive to wacc and terminal_growth — small changes move the result a lot. That's why every response includes the sensitivity grid by default: read the range it spans, not just the point estimate. In the example above, ±1% of WACC moves the per-share value from $96.60 to $134.04.

Disclaimer

Valuations returned by this API are model outputs computed from your assumptions and third-party financial data. They are estimates for analysis, not investment advice, recommendations, or offers to buy or sell any security. Verify important figures against primary sources before relying on them.
